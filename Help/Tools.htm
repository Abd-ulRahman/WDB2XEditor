Defintion Editor

The definition editor allows the ability to create, edit and delete defintions.

These can also be created and edited manually. Any *.xml files in the Definition will be imported provided they meet the basic rules of being named correctly and having a single int32 ID column.

Available Types:

  • byte
  • float
  • int
  • loc - This is for pre-Cata strings whereby the DBC file contains localisations and a mask
  • long
  • sbyte
  • short
  • string
  • uint
  • ulong
  • ushort

There are also a number of additional settings that are rarely used.

  • AutoGenerated - This is used to add a column that does not read data from the file i.e. CharBaseInfo where there is no ID field
  • ArraySize - Allows the grouping of simlar fields. The column name will be incremented when displayed i.e. Field, Field_1, Field_2 etc
  • Padding - This is used where files are nonuniformly padded such as having a padding in the middle of the file
